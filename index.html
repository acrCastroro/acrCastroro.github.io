N.A.N

Local multiplayer player vs player 2,5D

N.A.N is a game based in titles such as Duck Game or Stick Fight, where the main objective is to have a great time with three more colleagues during a house meeting, transporting us to the best way possible to play: physically together in the same room like in the old times.

It was made only by programmers in Unreal Engine 4, so all the art of the game is basically geometries and some free assets from Internet. During the implementation of this videogame the team was focused in end with a dynamic and easy to learn product, where the fights do not last more than a minute and a half. This will help to the user to play other and other and another match, feeling a potent impulse to continue playing more than one complete game (fifteen minutes approximately games).

My contributions:

  • Mechanics Programmer. Focus in the character movement and dodge, make it simple but extremely fun. Also the creator of a platformer system, designed for designers (forgive the redundancy)

  • Prototype Developer. Preparing all the essential tasks to have as soon as possible some product to iterate and polish mechanics.

  • PR programmer. Person in charge of create the presentations and documentations while the development period.

Rendering engine

Multithreading rendering engine done with C++ and OpenGL.

Biguili Engine is a compoent based multithreading rendering engine made in our last years of HND in Valencia in C++ from scratch, using the OpenGL 3.3 api.


My contribution

  • Component Based System. Following a hierachy of Game Objects based in the Unity one. Having components to divide and simplify the user tool management, creating functions to easily modiffy the component values, such as it texture, size, position or color.

  • Buffer System.Organized and easily fast access data containers with all the relevant information that the engine will need to work properly, baking the bug procces easier.

  • Display List System. A way to comunicate CPU and GPU using hidden commands (for the user) wich allows the simultaneous operation of these components.

  • Obj Loader integration. Incorporation of the library TinyObjLoader to the project for be able to use .obj files.

  • Light System. Generation of the Phong reflection model as basic lighting of the engine. Multe lights can affect to the same object.

  • Shadow Mapping. Integration of multiple shadows per object in the same scene.

  • Scripting support integration. Incorporation of the library ChaiScript.

  • Smart pointers integration. With the intention of stay safe the memory of the user hands.
